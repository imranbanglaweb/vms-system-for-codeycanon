Installation Guide

Step 1: Download and Setup

Download the package and extract it to your web server directory (e.g., public_html or htdocs).

Step 2: Install Dependencies

Open your terminal and navigate to the project directory:

cd /path/to/your/project
composer install
npm install

Step 3: Configure Environment

Copy the example environment file:

cp .env.example .env

Edit the .env file with your database credentials:

APP_NAME="Codeaycanon TMS"
APP_URL=http://yourdomain.com

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_database_username
DB_PASSWORD=your_database_password

Step 4: Generate Application Key

php artisan key:generate

Step 5: Run Database Migrations

php artisan migrate --seed

Step 6: Create Storage Link

php artisan storage:link

Step 7: Set File Permissions

chmod -R 775 storage/
chmod -R 775 bootstrap/cache/
Note: The permissions (775) may need to be adjusted based on your server configuration. Some servers require 755.
Success! Your application should now be accessible at http://yourdomain.com

Multi-Level Approval System

InayaFleet360 implements a comprehensive multi-level approval workflow for transport requisitions:

Approval Hierarchy

  1. Employee submits transport requisition
  2. Department Head receives notification and reviews (First Approval Level)
  3. Transport Admin receives notification and provides final approval (Final Approval Level)
  4. System generates trip sheet and assigns vehicle/driver

User Roles in Approval Workflow

Employee Employee Login

Employees can:

  • Submit transport requisitions
  • View requisition history and status
  • Track trip progress
  • Cancel pending requisitions

Driver Driver Login

Drivers can:

  • View assigned trips
  • Access personal dashboard
  • Manage availability status
  • Log fuel consumption
  • View schedule and calendar

Dept Head Department Head Login

Department Heads have approval authority:

  • View all requisitions from department employees
  • Approve requisitions (first approval level)
  • Reject requisitions with reason
  • View department transport statistics
  • Receive email and push notifications for pending approvals

Transport Admin Transport Admin Login

Transport Admins have final approval authority:

  • View all approved requisitions from all departments
  • Approve for final authorization (final approval level)
  • Reject requisitions with reason
  • Assign vehicles and drivers to approved requisitions
  • Generate and manage trip sheets
  • View overall fleet utilization

Approval Status Tracking

Status Description
Pending Submitted, awaiting Department Head approval
Department Approved Approved by Department Head, awaiting Transport Admin
Transport Approved Fully approved, trip sheet can be generated
Rejected Rejected by Department Head or Transport Admin
Cancelled Cancelled by the requester
