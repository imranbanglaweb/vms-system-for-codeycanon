ISSUE: Accessing /metapixel/conversions returned 403 Unauthorized

ROOT CAUSE:
The MetaPixelController was using Spatie role middleware that required users
to have explicit Admin or Super Admin roles. The menu system also checked for
'analytics-view' and 'system-configure' permissions which were not assigned
to the Admin user.

FIXES APPLIED:

1. app/Http/Controllers/Admin/MetaPixelController.php
   - Removed role middleware (role:Admin|Super Admin)
   - Simplified to only use auth middleware
   - Now accessible to all authenticated users

2. database/seeders/MenuSeeder.php
   - Changed parent menu permission from 'analytics-view' to null
   - Changed all sub-menu permissions from 'analytics-view'/'system-configure' to null
   - Menu now visible to all authenticated users

3. database/seeders/UserSeeder.php
   - Added permission assignment for Admin role
   - Added permission assignment for Super Admin role
   - Both roles now have 'analytics-view' and 'system-configure' permissions
